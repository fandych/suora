# Suora 用户操作指南

本指南基于当前代码实现编写，面向日常使用者，重点说明 Suora 现在已经具备的用户功能、推荐操作顺序、常见工作流和排查方法，而不是历史规划或旧版设计稿中的能力。

文档清理后，`docs/user/USER_GUIDE_ZH.md` 与 `docs/user/USER_GUIDE_EN.md` 是仓库里长期维护的主用户文档；更细分的测试、渠道与产品边界说明分别放在 `docs/TESTING.md`、`docs/CHANNEL_INTEGRATION.md` 与 `docs/requirements.md`。

## 1. 产品定位

Suora（朔枢）是一个本地 AI 工作台。当前版本不是只有聊天窗口，而是由多个协作模块组成：聊天、文档、模型、Agent、技能、流水线、定时器、渠道、MCP 和设置。

你可以把它理解为“本地运行的多 Agent 操作界面”，适合做以下事情：

- 与不同模型进行日常问答和任务执行
- 通过专业 Agent 处理代码、写作、研究、安全、数据和运维任务
- 在本地维护知识文档，并把文档内容带入对话
- 把多个 Agent 步骤串成流水线，手动运行或定时运行
- 接入企业微信、飞书、钉钉等平台，让桌面助手从外部渠道接收消息
- 管理模型提供商、技能、MCP 服务和桌面安全策略

## 2. 安装与首次启动

### 2.1 环境要求

如果你使用发布包安装，只需要准备 Windows、macOS 或 Linux 桌面环境。

如果你从源码运行，建议使用与 CI 接近的环境：

- Node.js 22.x
- npm
- 可运行 Electron 桌面应用的系统环境

### 2.2 从发布包安装

如果你不是开发者，优先使用 GitHub Releases 中的桌面安装包：

1. 打开 <https://github.com/fandych/suora/releases/latest>
2. 根据操作系统选择安装包：
   - Windows：安装版或便携版
   - macOS：DMG 或 ZIP
   - Linux：AppImage、DEB 或 RPM
3. 安装完成后启动应用。
4. 首次进入后先配置至少一个模型提供商，否则聊天、Agent、流水线等能力无法正常调用模型。

### 2.3 从源码运行

开发者可以在仓库根目录执行：

```bash
npm install
npm run dev
```

如果依赖安装失败，先确认 Node.js 版本、npm 镜像、系统代理和本机网络是否正常。

### 2.4 首次引导

首次打开应用时会看到 5 步引导：

1. 欢迎
2. 配置模型提供商
3. 认识你的 Agent
4. 浏览技能
5. 准备完成

如果跳过引导，可以在 `Settings -> System` 里重新运行。

### 2.5 推荐首次配置顺序

为了尽快进入稳定可用状态，建议按下面顺序完成首次配置：

1. 在 **Models** 中添加一个云端提供商，或配置本地 Ollama 端点。
2. 在 **Models -> Models** 中启用需要使用的模型。
3. 在 **Chat** 中选择默认模型与默认 Agent。
4. 在 **Skills** 中确认已启用需要的技能。
5. 在 **Documents** 中创建至少一个文档组，建立你的本地知识区。
6. 如果要做自动化，再去 **Pipeline** 和 **Timer**。
7. 如果要接入外部消息平台，再配置 **Channels**。
8. 如果 Agent 需要外部工具或上下文协议能力，再配置 **MCP**。
9. 最后进入 **Settings** 检查语言、主题、安全、数据备份和日志选项。

## 3. 工作台总览

当前主导航对应的模块如下：

| 模块 | 当前用途 |
| --- | --- |
| Chat | 多会话对话、切换 Agent / 模型、发送附件、查看工具调用 |
| Documents | 管理本地文档组、文件夹、反向链接和图谱 |
| Pipeline | 设计与运行多步骤 Agent 流水线 |
| Models | 配置 provider、启用模型、测试连接、比较模型 |
| Agents | 管理内置和自定义 Agent，做测试、导入导出和版本快照 |
| Skills | 管理已安装技能、浏览注册表技能、编辑 `SKILL.md` |
| Timer | 创建单次、间隔和 Cron 定时任务 |
| Channels | 接入消息平台并指定回复 Agent |
| MCP | 配置 MCP 服务器 |
| Settings | 管理通用偏好、安全、语音、快捷键、数据、日志和系统诊断 |

当前默认跳转为：

- `/ -> /chat`
- `/models -> /models/providers`
- `/settings -> /settings/general`

### 3.1 推荐日常操作路径

- 普通问答：`Models` 配好模型后，直接进入 `Chat`。
- 知识问答：先在 `Documents` 整理资料，再在 `Chat` 引入文档上下文。
- 专业任务：先在 `Agents` 选择或创建 Agent，再回到 `Chat` 执行任务。
- 重复流程：先在 `Pipeline` 编排步骤，再从 UI、聊天命令或 `Timer` 触发。
- 外部消息：先在 `Channels` 配置平台与回复 Agent，再查看健康状态和消息历史。
- 工具扩展：在 `MCP` 添加服务器配置，再让 Agent 使用对应能力。

## 4. Chat：聊天与任务执行

聊天页是当前最核心的操作入口，已经实现的能力包括：

- 多会话列表与标签
- 会话级 Agent 与模型切换
- 图片、文件、音频附件
- 流式回复
- Markdown、代码块、数学公式显示
- 工具调用列表和执行状态
- 回复失败后一键重试
- 消息编辑、删除、置顶、分支对话
- 回复反馈（赞 / 踩）
- 文本朗读
- 行内引用展示

### 4.1 创建一次基础对话

1. 打开左侧导航中的 **Chat**。
2. 新建或选择一个会话。
3. 在会话顶部或侧边控制区选择 Agent。
4. 选择本次会话要使用的模型。
5. 在输入框输入问题或任务说明。
6. 按 `Enter` 发送；如果需要换行，按 `Shift + Enter`。
7. 等待流式回复完成后，可以继续追问、编辑消息、重试失败回复或给回复反馈。

### 4.2 选择 Agent 与模型

建议按任务类型选择：

- 普通问答、解释概念：使用 Assistant。
- 写作、改写、文案：使用 Writing Strategist。
- 代码分析、代码生成：使用 Code Expert。
- 资料整理、调研摘要：使用 Research Analyst。
- 安全检查：使用 Security Auditor。
- 数据整理、表格分析：使用 Data Analyst。
- 运维、脚本、部署相关问题：使用 DevOps Expert。
- 想让系统帮你设计新 Agent：使用 Agent builder。
- 想生成或调整流水线：使用 Pipeline builder。
- 想生成定时任务方案：使用 Timer builder。
- 想编辑或整理文档内容：使用 Document editor。

模型选择建议：

- 复杂推理、长文档总结：选择能力更强、上下文更大的模型。
- 快速问答、草稿生成：选择响应更快、成本更低的模型。
- 本地隐私优先：优先使用 Ollama 等本地模型。
- 需要兼容私有网关：使用 OpenAI-compatible 类型的提供商。

### 4.3 使用附件和文档上下文

聊天支持图片、文件和音频附件。使用时建议：

1. 先确认当前模型支持对应输入类型。
2. 上传附件后，在消息里说明你希望模型如何处理附件。
3. 如果是长期知识资料，优先整理到 **Documents**，再作为文档上下文使用。
4. 对包含敏感信息的文件，先检查 `Settings -> Security` 中的文件系统和工具执行策略。

### 4.4 处理回复结果

收到回复后可以进行以下操作：

- 继续追问：保留当前上下文，让模型基于已有对话继续处理。
- 编辑上一条用户消息：修正提示词后重新生成。
- 重试失败回复：适合网络抖动、模型限流或临时错误。
- 分支对话：尝试不同方案时，把当前会话拆出分支。
- 置顶关键消息：保留重要结论，方便之后回看。
- 查看工具调用：如果 Agent 调用了工具，可以查看调用状态和结果。

### 4.5 当前可直接使用的快捷操作

- `Ctrl/Cmd + K`：打开命令面板
- `Enter`：发送消息
- `Shift + Enter`：输入框换行
- `Escape`：关闭命令面板或弹窗
- `Ctrl/Cmd + S`：在文档编辑器里保存内容

### 4.6 命令面板

命令面板可以快速跳转到：

- 会话
- 文档
- Agent
- 技能
- 模型
- 设置
- 渠道
- 定时器
- MCP
- Pipeline

## 5. Models：模型与提供商

模型模块用于管理模型提供商、具体模型和连接测试。建议先完成这一部分，再使用聊天、Agent 和流水线。

### 5.1 当前运行时支持的提供商

当前运行时提供商支持包括：

- Anthropic
- OpenAI
- Google
- Ollama
- DeepSeek
- Zhipu
- MiniMax
- Groq
- Together AI
- Fireworks
- Perplexity
- Cohere
- OpenAI 兼容端点

当前 UI/provider 预设和运行时支持不是完全同一层概念：模型配置里会种子化 OpenAI、Anthropic、Google Gemini、Ollama、DashScope、Kimi 和通用 OpenAI-compatible 等预设；提供商编辑器当前暴露的 provider 类型选择主要包括 OpenAI、Anthropic、Google、Ollama 和 OpenAI-compatible。

### 5.2 添加提供商

1. 打开 **Models -> Providers**。
2. 点击新建或添加提供商。
3. 选择合适的 provider 类型或预设。
4. 填写显示名称，便于在聊天和 Agent 中识别。
5. 填写 API Key。
6. 如需自定义网关或私有部署，填写 Base URL。
7. 保存配置。
8. 点击测试连接，确认提供商可用。

如果使用 Ollama，本地端点默认走 `http://localhost:11434/v1`。使用前需要先在本机启动 Ollama 服务，并拉取至少一个模型。

### 5.3 启用模型

1. 打开 **Models -> Models**。
2. 找到刚配置的提供商。
3. 勾选或启用要在工作台中使用的模型。
4. 根据需要设置 `temperature` 和 `maxTokens`。
5. 回到 Chat 或 Agent 编辑器，选择已启用的模型。

建议只启用实际使用的模型，避免下拉列表过长。

### 5.4 比较模型

在 **Models -> Compare** 中可以对比可用模型。适合以下场景：

- 评估同一提示词在不同模型中的输出质量。
- 比较速度、稳定性和格式遵循能力。
- 为 Agent 或流水线选择默认模型。

## 6. Agents：内置与自定义 Agent

Agent 是带有角色、提示词、模型、技能、工具权限和行为配置的执行单元。它不是简单的提示词模板。

### 6.1 当前内置 Agent

当前内置 Agent 包括：

- Assistant
- Agent builder
- Pipeline builder
- Timer builder
- Document editor
- Code Expert
- Writing Strategist
- Research Analyst
- Security Auditor
- Data Analyst
- DevOps Expert

### 6.2 选择合适的 Agent

- 不确定使用哪个：先用 Assistant。
- 需要构建新 Agent：使用 Agent builder 描述目标角色和边界。
- 需要生成自动化流程：使用 Pipeline builder。
- 需要定时提醒或周期执行：使用 Timer builder。
- 需要整理、改写、扩充文档：使用 Document editor。
- 需要写代码、查错、解释实现：使用 Code Expert。
- 需要文章、报告、邮件、营销文案：使用 Writing Strategist。
- 需要资料检索、信息归纳、研究计划：使用 Research Analyst。
- 需要威胁建模、风险排查、安全建议：使用 Security Auditor。
- 需要数据整理、指标解释、分析结论：使用 Data Analyst。
- 需要命令、部署、监控、运维方案：使用 DevOps Expert。

### 6.3 创建自定义 Agent

1. 打开 **Agents**。
2. 新建 Agent 或复制现有 Agent。
3. 填写名称、头像和颜色。
4. 编写系统提示词，明确它的角色、边界、输出格式和禁止事项。
5. 绑定默认模型。
6. 选择需要的技能。
7. 设置温度、最大轮次、回复风格等行为参数。
8. 配置允许 / 禁止工具列表。
9. 根据需要开启自动学习。
10. 在 Agent 页内测试对话。
11. 确认效果稳定后保存。

### 6.4 版本、复制、导入与导出

Agent 页支持：

- 导入 Agent
- 导出 Agent
- 复制现有 Agent
- 版本快照
- 从快照恢复
- 在页面内直接测试

建议在大幅修改提示词、工具权限或技能绑定前创建版本快照，方便回滚。

## 7. Skills：技能管理

技能系统以 `SKILL.md` 为核心，主要用于给 Agent 提供可复用的能力说明、流程约束和资源入口。技能不是内置工具注册表本身。

### 7.1 技能页当前支持

- 查看已安装技能
- 启用或禁用技能
- 编辑 `SKILL.md`
- 浏览技能注册表
- 安装注册表技能前查看安装预览
- 添加 / 管理技能源
- 从单个文件导入技能
- 从整个技能目录导入技能
- 导出技能为 markdown 或 zip

技能还支持从工作区和外部目录自动加载。

### 7.2 安装和启用技能

1. 打开 **Skills**。
2. 在已安装列表中查看当前技能状态。
3. 如果技能已存在，直接启用。
4. 如果需要新技能，进入注册表浏览或导入本地技能文件 / 目录。
5. 安装前查看预览，确认技能说明和资源内容符合预期。
6. 安装后返回 Agent 设置，把技能绑定到需要使用它的 Agent。

### 7.3 编辑 `SKILL.md`

编辑技能时建议包含：

- 技能名称和用途。
- 适用场景。
- Agent 应遵守的流程。
- 输入要求和输出格式。
- 注意事项、限制和安全边界。
- 必要的参考资源说明。

修改后建议用绑定该技能的 Agent 进行一次测试对话，确认技能描述能被正确执行。

### 7.4 导入、导出和外部目录

- 单文件导入适合只有一个 `SKILL.md` 的简单技能。
- 目录导入适合包含资源文件的技能包。
- 导出 markdown 适合分享文本说明。
- 导出 zip 适合迁移完整技能包。
- 外部目录加载适合维护个人或团队技能库。

## 8. Documents：文档知识区

文档模块用于维护本地知识内容，并把内容带入聊天、Agent 或后续自动化流程。

### 8.1 文档页当前支持

- 文档组
- 嵌套文件夹
- Markdown 文档
- Mermaid 图表
- 数学公式块
- 文档搜索
- 反向链接和引用关系
- 基于来源信息扩展相关笔记
- 图谱视图，并提示桥接节点、稀疏簇、知识缺口和意外连接
- 把文档选为聊天上下文

### 8.2 建立文档结构

推荐按下面方式组织：

1. 先按主题创建文档组，例如项目、客户、研究方向或知识领域。
2. 在组内用文件夹拆分子主题、阶段或资料类型。
3. 每个文档只聚焦一个相对清晰的问题或主题。
4. 用标题、标签和链接维护关联。
5. 定期查看图谱和反向链接，发现孤立文档、知识缺口和意外关联。

### 8.3 编写和保存文档

1. 打开 **Documents**。
2. 选择文档组和文件夹。
3. 新建 Markdown 文档。
4. 输入标题和正文。
5. 如需图表，使用 Mermaid 语法块。
6. 如需公式，使用数学公式块。
7. 使用 `Ctrl/Cmd + S` 保存。
8. 保存后可在搜索、图谱和聊天上下文中继续使用。

### 8.4 把文档带入聊天

1. 在 Documents 中整理并保存资料。
2. 打开 Chat。
3. 选择本次任务需要的 Agent 和模型。
4. 选中相关文档作为上下文。
5. 在问题中说明你希望模型如何使用这些文档，例如“只基于这些文档总结风险点”或“根据文档生成执行清单”。
6. 如果回答不完整，继续补充范围、输出格式或引用要求。

## 9. Pipeline：流水线自动化

Pipeline 用于把多个 Agent 步骤串成可重复执行的流程，适合处理固定格式、跨角色、多阶段的任务。

### 9.1 流水线页当前支持

- 多步骤 Agent 编排
- 步骤重试和退避策略
- 步骤超时
- 条件运行 `runIf`
- 输出裁剪与变量导出
- 预算限制：总时长、总 Token、最大执行步数
- Mermaid 预览与源码导出
- 执行历史与步骤详情
- 保存、导入、导出流水线

### 9.2 创建流水线

1. 打开 **Pipeline**。
2. 新建流水线并填写名称、说明和使用场景。
3. 添加第一个步骤，选择负责执行的 Agent。
4. 编写该步骤的输入提示词。
5. 如后续步骤需要使用当前步骤结果，配置输出变量。
6. 继续添加后续步骤，并说明每一步如何使用前一步输出。
7. 根据需要设置 `runIf` 条件、超时、重试次数和退避策略。
8. 设置整体预算，例如最大执行时长、最大 Token 或最大步骤数。
9. 查看 Mermaid 预览，确认流程结构无误。
10. 保存并试运行。
11. 查看执行历史和步骤详情，修正提示词或条件。

### 9.3 从聊天触发流水线

聊天中支持 `/pipeline` 相关命令，例如：

- `/pipeline list`：列出可用流水线
- `/pipeline run <名称或ID>`：运行指定流水线
- `/pipeline status`：查看当前状态
- `/pipeline history <名称或ID>`：查看历史
- `/pipeline cancel`：取消正在运行的流水线

适合在对话中把临时需求转成自动化执行。

### 9.4 流水线设计建议

- 每个步骤只做一类事情，例如“提取事实”“生成草稿”“检查风险”。
- 步骤输出要结构化，方便后续步骤引用。
- 对依赖外部工具的步骤设置超时和重试。
- 对成本敏感的流程设置总 Token 和最大步骤数。
- 大改之前先导出备份。

## 10. Timer：定时任务

Timer 用于按时间触发提醒、Agent Prompt 或已保存流水线。

### 10.1 当前定时器类型

- Once：只执行一次。
- Interval：按固定间隔重复执行。
- Cron：按 Cron 表达式执行。

### 10.2 当前定时器动作

- 桌面通知
- 触发 Agent Prompt
- 触发已保存流水线

### 10.3 创建定时器

1. 打开 **Timer**。
2. 新建定时任务。
3. 选择执行类型：Once、Interval 或 Cron。
4. 设置触发时间、间隔或 Cron 表达式。
5. 选择动作：通知、Agent Prompt 或流水线。
6. 如果选择 Agent Prompt，选择 Agent、模型和提示词。
7. 如果选择流水线，选择已保存的流水线，并确认输入参数。
8. 保存后检查任务是否启用。
9. 等待触发后查看执行结果和通知。

### 10.4 使用建议

- 重要提醒使用 Once 或 Cron。
- 高频任务使用 Interval 时要控制间隔，避免频繁调用模型。
- 自动化任务优先封装为 Pipeline，再由 Timer 调度。
- 定时任务涉及文件、工具或外部平台时，先检查安全设置和 Agent 权限。

## 11. Channels：外部消息渠道

Channels 用于接入外部消息平台，并把入站消息路由给指定 Agent 回复。

### 11.1 当前支持的渠道平台

- 企业微信
- 微信公众号
- 微信小程序
- 飞书 / Lark
- 钉钉
- Slack
- Telegram
- Discord
- Microsoft Teams
- 自定义渠道

### 11.2 渠道页当前支持

- Webhook 或 Stream 模式
- 绑定一个回复 Agent
- 自动回复开关
- 白名单聊天对象
- 查看消息历史
- 查看用户列表
- 查看健康状态
- 查看调试信息

### 11.3 创建渠道

1. 打开 **Channels**。
2. 新建渠道并选择平台。
3. 选择 Webhook 或 Stream 模式。
4. 填写平台要求的密钥、Token、URL 或其他连接参数。
5. 绑定负责回复的 Agent。
6. 根据需要开启或关闭自动回复。
7. 配置白名单聊天对象，限制允许触发回复的来源。
8. 保存并启用渠道。
9. 在平台侧配置回调地址或连接参数。
10. 回到 Suora 的 Health / Debug 面板检查状态。
11. 发送测试消息，确认消息历史和回复流程正常。

### 11.4 使用建议

- 生产渠道优先绑定经过测试的 Agent。
- 对外回复前，先关闭自动回复做测试，确认输出稳定后再开启。
- 白名单应尽量收窄，避免无关群聊或用户触发模型调用。
- 出现平台回调失败时，同时检查平台后台、Suora Debug 面板和本机网络。

## 12. MCP：模型上下文协议服务

MCP 模块用于配置 MCP 服务器，并把外部能力纳入 Agent 可用范围。

### 12.1 MCP 页当前用途

- 添加服务器配置
- 编辑服务器配置
- 查看连接状态
- 把 MCP 服务纳入 Agent 可用能力范围

### 12.2 添加 MCP 服务器

1. 打开 **MCP**。
2. 新建服务器配置。
3. 填写服务器名称、启动方式或连接参数。
4. 保存配置。
5. 查看连接状态。
6. 如果连接失败，检查命令、环境变量、工作目录和权限。
7. 在 Agent 配置中允许使用对应 MCP 能力。
8. 回到 Chat 或 Agent 测试对话，确认能力可用。

### 12.3 安全建议

- 只添加可信 MCP 服务器。
- 不要把敏感目录或高风险命令无差别暴露给 Agent。
- 对需要文件系统或 shell 能力的 MCP 服务，先检查 `Settings -> Security`。
- 发现异常调用时，先禁用相关服务器或收紧 Agent 工具权限。

### 13.1 当前设置分区

当前设置页包含 7 个分区：

- General
- Security
- Voice
- Shortcuts
- Data
- Logs
- System

### 13.2 General

常用配置包括：

- 主题
- 语言
- 字体
- 强调色
- 自动启动
- 工作区默认偏好
- 代理服务器配置
- SMTP 邮件配置与测试
- 环境变量管理

建议首次使用时先检查语言、主题、代理和默认工作区配置。

### 13.3 Security

重要安全能力包括：

- API Key 管理
- 工具执行确认
- 文件系统沙箱模式
- 允许目录列表
- 危险命令屏蔽模式
- 隐私和安全默认值

API Key 会优先写入系统安全存储；如果系统 Keyring 不可用或加密失败，应用会提示这些 Key 只保存在内存中，重启后需要重新输入。

### 13.4 Voice

Voice 分区用于管理：

- 语音输入
- 语音输出
- 音频行为

- 面板控制快捷键
- 文档编辑保存等操作

### 13.6 Data

Data 分区用于管理：

- 数据导入与导出

### 13.7 Logs

Logs 分区用于查看：

- 运行日志
- 崩溃记录
- 调试信息
- 问题排查证据

### 13.8 System

System 分区用于：

- 重新运行首次引导
- 查看应用健康状态
- 查看运行时性能指标
- 检查系统诊断信息

如果应用行为异常，先查看 System 和 Logs，再决定是否重启应用或清理配置。

## 14. 常见使用路径

### 14.1 场景 1：完成一次普通问答

1. 在 Models 中确认至少一个模型可用。
2. 打开 Chat。
3. 选择 Assistant 和目标模型。
4. 输入问题。
5. 根据回答继续追问、重试或分支。

### 14.2 场景 2：把文档知识带入对话

1. 在 Documents 中创建文档组。
2. 建立文件夹并整理资料。
3. 保存 Markdown 文档。
4. 打开 Chat，选择当前会话的 Agent 与模型。
5. 选中相关文档作为上下文。
6. 提问、总结、改写或继续执行任务。

### 14.3 场景 3：创建一个专业 Agent

1. 打开 Agents。
2. 复制类似的内置 Agent，或新建一个 Agent。
3. 写清楚角色、任务边界、输出格式和禁止事项。
4. 绑定模型和技能。
5. 配置工具允许 / 禁止列表。
6. 在 Agent 页内测试。
7. 创建版本快照。
8. 回到 Chat 使用该 Agent。

### 14.4 场景 4：把重复任务做成流水线

1. 在 Pipeline 中创建多步骤流程。
2. 为每一步选择合适的 Agent。
3. 设置 `runIf`、超时、重试和输出变量。
4. 试运行并查看步骤详情。
5. 保存后可以从聊天 `/pipeline run` 触发，或交给 Timer 定时执行。

### 14.5 场景 5：定时运行一个流程

1. 先在 Pipeline 中保存可运行的流水线。
2. 打开 Timer。
3. 新建 Cron 或 Interval 任务。
4. 选择“触发已保存流水线”。
5. 配置执行时间和输入参数。
6. 保存并启用。
7. 到时间后查看通知和流水线执行历史。

### 14.6 场景 6：接入外部消息渠道

1. 在 Channels 中创建渠道。
2. 选择平台和连接模式。
3. 绑定负责回复的 Agent。
4. 填写平台密钥、Webhook 或 Stream 参数。
5. 配置白名单聊天对象。
6. 在 Health / Debug 面板里检查运行状态。
7. 发送测试消息确认消息历史和回复结果。

### 14.7 场景 7：让 Agent 使用外部 MCP 能力

1. 在 MCP 中添加服务器配置。
2. 确认连接状态正常。
3. 在 Agent 中允许对应能力。
4. 回到 Chat 选择该 Agent。
5. 用一个可验证的小任务测试能力是否可用。
6. 如果失败，检查 MCP 状态、Agent 工具权限和安全设置。

## 15. 常见问题排查

### 15.1 模型没有回复或报连接错误

- 先检查提供商的 API Key、Base URL 和网络连通性。
- 如果是 Ollama，确认本地端点是否在运行。
- 在 Models 中重新做连接测试。
- 检查模型是否已在 Models 中启用。
- 如果使用公司代理或私有网关，检查 `Settings -> General` 中的代理配置。

### 15.2 聊天能打开，但 Agent 输出不符合预期

- 检查当前会话选择的 Agent 是否正确。
- 检查 Agent 的系统提示词、技能绑定和模型绑定。
- 降低或调整温度，提升输出稳定性。
- 对复杂任务拆成多轮输入，或改用 Pipeline。
- 如果涉及工具调用，检查 Agent 的允许 / 禁止工具列表。

### 15.3 重启后 API Key 丢失

- 这通常说明系统 Secure Storage / Keyring 不可用。
- Suora 会在这种情况下只把密钥保存在内存里，重启后需要重新输入。
- 可以检查系统钥匙串、凭据管理器或 Linux keyring 服务是否可用。

### 15.4 文档搜索不到内容

- 确认文档已经保存。
- 检查关键词是否出现在标题、路径、标签、标题层级或正文中。
- 尝试使用更短、更明确的关键词。
- 如果刚导入大量文档，等待索引刷新后再试。

### 15.5 流水线执行失败

- 查看执行历史和失败步骤详情。
- 检查步骤绑定的 Agent 是否仍然存在。
- 检查模型和提供商是否可用。
- 检查 `runIf` 条件、输出变量名和后续引用是否一致。
- 增加超时时间或降低步骤复杂度。
- 对不稳定步骤配置重试和退避。

### 15.6 定时器没有触发

- 确认定时器已启用。
- 检查 Once、Interval 或 Cron 配置是否正确。
- 确认应用在触发时间处于运行状态。
- 如果动作是 Agent Prompt 或 Pipeline，确认模型、Agent 和流水线仍然可用。
- 查看 Logs 和 System 获取诊断信息。

### 15.7 渠道收不到消息

- 检查渠道是否启用。
- 检查 reply agent 是否仍然存在。
- 检查 Webhook 地址、签名密钥和白名单聊天对象。
- 检查平台后台是否成功发送回调。
- 在 Channels 的 Health / Debug 面板查看状态和错误信息。

### 15.8 MCP 服务不可用

- 检查 MCP 服务器配置是否保存。
- 检查启动命令、工作目录和环境变量。
- 检查连接状态。
- 检查 Agent 是否允许使用对应能力。
- 检查安全设置是否阻止了相关文件或命令访问。

### 15.9 看不到某个功能入口

- 先确认当前路由是否存在于主导航。
- `Models`、`Skills`、`Settings` 里有子视图与默认跳转，不是所有入口都在顶层按钮直接展开。
- 可以使用 `Ctrl/Cmd + K` 打开命令面板快速跳转。

## 16. 使用建议与安全边界

### 16.1 提示词建议

- 说明背景：告诉 Agent 任务上下文和目标。
- 明确输出：指定表格、清单、步骤、JSON 或 Markdown 等格式。
- 给出边界：说明不要做什么、不能访问什么、不能假设什么。
- 拆分任务：复杂任务拆成“分析 -> 生成 -> 检查 -> 修改”。
- 保留证据：需要审计时要求列出依据、引用或决策理由。

### 16.2 数据安全建议

- 不要把不必要的敏感信息发给云端模型。
- 处理私密资料时，优先考虑本地模型或受控私有端点。
- 给 Agent 开放文件系统、shell 或 MCP 能力前，先确认安全设置。
- 使用渠道自动回复时，配置白名单并先测试输出。
- 大规模清理、导入、导出前先备份。

### 16.3 自动化建议

- 先用 Chat 验证提示词，再固化为 Agent。
- 先用 Agent 验证单步能力，再组合成 Pipeline。
- 先手动运行 Pipeline，再交给 Timer 定时执行。
- 对外部渠道，先手动测试，再开启自动回复。
- 定期查看 Logs、System 和执行历史，避免自动化静默失败。
