# دليل مستخدم Suora

هذا الدليل مبني على التنفيذ الحالي في الشفرة المصدرية. وهو يصف ما يستطيع Suora القيام به الآن، وليس ما ظهر في وثائق قديمة أو خطط سابقة.

## 1. ما هو Suora

Suora عبارة عن منصة عمل محلية للذكاء الاصطناعي. التطبيق الحالي ليس مجرد نافذة دردشة، بل مساحة عمل مكتبية متعددة الوحدات تضم الدردشة والمستندات والنماذج والوكلاء والمهارات وخطوط الأنابيب والمؤقتات والقنوات وخوادم MCP والإعدادات.

يمكنك استخدامه من أجل:

- إجراء محادثات يومية وتنفيذ مهام باستخدام نماذج مختلفة
- توجيه العمل إلى وكلاء متخصصين للبرمجة والكتابة والبحث والأمن والبيانات وعمليات DevOps
- حفظ مساحة مستندات محلية وإرفاق سياق المستندات بالمحادثات
- إنشاء خطوط أنابيب متعددة الخطوات وتشغيلها يدويًا أو بجدول زمني
- ربط منصات الرسائل الخارجية حتى يتمكن المساعد المكتبي من الرد على الرسائل الواردة

## 2. التثبيت وأول تشغيل

### المتطلبات

- بيئة سطح مكتب على Windows أو macOS أو Linux
- Node.js 18+ عند التشغيل من المصدر
- npm

### التشغيل من المصدر

```bash
npm install
npm run dev
```

### التهيئة الأولى

عند أول تشغيل، يعرض Suora مسار تهيئة من خمس خطوات:

1. Welcome
2. Configure a Model Provider
3. Meet Your Agents
4. Explore Skills
5. You're All Set

إذا تخطيت هذا المسار، يمكنك تشغيله مرة أخرى من `Settings -> System`.

## 3. خريطة مساحة العمل

| الوحدة | الاستخدام الحالي |
| --- | --- |
| Chat | محادثات متعددة الجلسات، تبديل الوكيل أو النموذج، المرفقات، ومتابعة استدعاءات الأدوات |
| Documents | مجموعات مستندات محلية، مجلدات، روابط خلفية، وعرض بياني |
| Pipeline | تصميم وتنفيذ سير عمل متعدد الخطوات للوكلاء |
| Models | إعداد المزودين، تفعيل النماذج، اختبار الاتصال، والمقارنة |
| Agents | إدارة الوكلاء المضمنين والمخصصين، الاختبار، الاستيراد والتصدير، والإصدارات |
| Skills | المهارات المثبتة، استعراض السجل، وتحرير `SKILL.md` |
| Timer | الجداول مرة واحدة، والفترات، وCron |
| Channels | تكاملات منصات الرسائل وتوجيه الردود |
| MCP | إعداد خوادم Model Context Protocol |
| Settings | التفضيلات والأمان والبيانات والسجلات والتشخيص |

## 4. سير عمل الدردشة

تجربة الدردشة الحالية تتضمن:

- جلسات وعلامات تبويب متعددة
- اختيار الوكيل والنموذج لكل جلسة
- مرفقات للصور والملفات والصوت
- ردود متدفقة
- عرض Markdown والشفرة والرياضيات
- عرض حالات استدعاء الأدوات
- إعادة المحاولة عند فشل الرد
- تعديل الرسائل وحذفها وتثبيتها وتشعيبها
- تقييم ردود المساعد
- القراءة الصوتية لردود المساعد
- إظهار الاقتباسات داخل الرسالة

### الاختصارات المتاحة الآن

- `Ctrl/Cmd + K` لفتح لوحة الأوامر
- `Enter` لإرسال الرسالة
- `Shift + Enter` لسطر جديد داخل حقل الإدخال
- `Escape` لإغلاق لوحة الأوامر أو النوافذ
- `Ctrl/Cmd + S` للحفظ داخل محرر المستندات

### لوحة الأوامر

يمكن أن تنتقل مباشرة إلى:

- الجلسات
- المستندات
- الوكلاء
- المهارات
- النماذج
- الإعدادات
- القنوات
- المؤقتات
- MCP
- Pipeline

### ما الذي تدعمه صفحة Models حاليًا

## 7. المستندات وخطوط الأنابيب والمؤقتات

### المستندات

تدعم وحدة Documents حاليًا:

- مجموعات المستندات
- المجلدات المتداخلة
- مستندات markdown
- مخططات Mermaid
- كتل رياضية
- البحث في المستندات
- الروابط الخلفية والمراجع
- العرض البياني
- استخدام المستندات المختارة كسياق للدردشة

### Pipeline

تدعم وحدة Pipeline حاليًا:

- سير عمل متعدد الخطوات للوكلاء
- إعادة المحاولة واستراتيجيات التراجع
- مهلات لكل خطوة
- التنفيذ الشرطي عبر `runIf`
- تحويلات المخرجات والمتغيرات المصدرة
- حدود لمدة التنفيذ وإجمالي التوكنات وعدد الخطوات
- معاينة Mermaid وتصدير المصدر
- سجل التنفيذ وتفاصيل الخطوات
- الحفظ والاستيراد والتصدير

كما تدعم الدردشة أوامر `/pipeline` مثل:

- `/pipeline list`
- `/pipeline run <name-or-id>`
- `/pipeline status`
- `/pipeline history <name-or-id>`
- `/pipeline cancel`

### Timer

أنواع المؤقتات الحالية هي:

- Once
- Interval
- Cron

والإجراءات الحالية هي:

- إشعار على سطح المكتب

## 10. استكشاف الأخطاء

### فشل اتصال النموذج

تحقق بالترتيب من:

1. صحة مفتاح API
2. توافق Base URL مع المزود
3. وجود نموذج مفعّل واحد على الأقل
4. عدم حظر الطلب بواسطة إعدادات الوكيل الشبكي
5. نجاح اختبار الاتصال في صفحة Models

### القناة لا تستقبل رسائل

تحقق بالترتيب من:

1. أن القناة مفعلة
2. أن وكيل الرد موجود ومفعل
3. أن خادم القنوات المحلي يعمل عند استخدام webhook
4. أن عنوان callback في المنصة يطابق عنوان Suora تمامًا
5. أن المحادثة الحالية ليست محجوبة بواسطة `allowedChats`
6. أن لوحتي Health وDebug لا تعرضان خطأ اعتماد

### المهارة لا تبدو فعالة

تحقق بالترتيب من:

1. أن المهارة مفعلة
2. أن الوكيل يملك المهارة المطلوبة
3. أن المهارة استوردت إلى مساحة العمل الحالية أو إلى دليل خارجي
4. أن محتوى المهارة صالح كملف `SKILL.md`

### المؤقت لا يعمل

تحقق بالترتيب من:

1. أن المؤقت مفعل
2. أن تعبير Cron صالح
3. أن الوكيل أو الـ Pipeline الهدف ما زال موجودًا
4. أن التطبيق المكتبي يعمل

## 11. الجلسة الأولى المقترحة

إذا كنت جديدًا على هذا الإصدار، فهذا التسلسل مناسب:

1. أضف مزودًا وفعّل نموذجًا في `Models`
2. راجع الوكلاء المضمنين في `Agents`
3. ابدأ أول محادثة في `Chat`
4. أنشئ مجموعة مستندات في `Documents`
5. احفظ سير عمل من خطوتين أو ثلاث في `Pipeline`
6. قم بجدولته من `Timer`
7. اضبط `Channels` أو `MCP` بعد استقرار سير العمل المحلي
