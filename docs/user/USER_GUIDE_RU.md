# Руководство пользователя Suora

Это руководство основано на текущей реализации в кодовой базе. Оно описывает то, что Suora умеет сейчас, а не то, что было указано в старых планах или устаревшей документации.

## 1. Что такое Suora

Suora — это локальная AI-workbench. Текущее приложение — не просто окно чата, а настольное рабочее пространство с модулями Chat, Documents, Models, Agents, Skills, Pipeline, Timer, Channels, MCP и Settings.

Suora можно использовать для того, чтобы:

- вести повседневные диалоги и выполнять задачи с разными моделями
- передавать работу специализированным агентам для кода, письма, исследований, безопасности, данных и DevOps
- поддерживать локальное пространство документов и подключать этот контекст к чатам
- собирать многошаговые пайплайны агентов и запускать их вручную или по расписанию
- подключать внешние мессенджеры, чтобы настольный помощник отвечал на входящие сообщения

## 2. Установка и первый запуск

### Требования

- настольная среда Windows, macOS или Linux
- Node.js 18+ при запуске из исходников
- npm

### Запуск из исходников

```bash
npm install
npm run dev
```

### Onboarding

При первом запуске Suora показывает пятишаговый onboarding:

1. Welcome
2. Configure a Model Provider
3. Meet Your Agents
4. Explore Skills
5. You're All Set

Если вы его пропустили, его можно снова запустить из `Settings -> System`.

## 3. Карта рабочего пространства

| Модуль | Текущее назначение |
| --- | --- |
| Chat | Многосессионный чат, переключение агентов и моделей, вложения и вызовы инструментов |
| Documents | Локальные группы документов, папки, обратные ссылки и графовое представление |
| Pipeline | Проектирование и запуск многошаговых workflows агентов |
| Models | Настройка провайдеров, включение моделей, тест соединения и сравнение |
| Agents | Управление встроенными и пользовательскими агентами, тестирование, импорт, экспорт и версии |
| Skills | Установленные skills, просмотр реестра и редактирование `SKILL.md` |
| Timer | Одноразовые, интервальные и Cron-расписания |
| Channels | Интеграции с мессенджерами и маршрутизация ответов |
| MCP | Настройка серверов Model Context Protocol |
| Settings | Настройки, безопасность, данные, логи и диагностика |

## 4. Работа с чатом

Текущий чат поддерживает:

- несколько сессий и вкладок
- выбор агента и модели для каждой сессии
- вложения изображений, файлов и аудио
- потоковые ответы
- рендеринг markdown, блоков кода и математики
- отображение статуса вызовов инструментов
- повторную отправку после ошибки
- редактирование, удаление, закрепление и ветвление сообщений
- оценку ответов ассистента
- озвучивание ответов ассистента
- встроенные цитаты

### Горячие клавиши, которые работают сейчас

- `Ctrl/Cmd + K`: открыть палитру команд
- `Enter`: отправить сообщение
- `Shift + Enter`: новая строка в поле ввода
- `Escape`: закрыть палитру или диалог
- `Ctrl/Cmd + S`: сохранить документ в редакторе

### Палитра команд

Через палитру можно быстро перейти к:

- сессиям
- документам
- агентам
- skills
- моделям
- настройкам
- каналам
- таймерам
- MCP
- pipeline

- markdown-документы

- графовое представление
- использование выбранных документов как контекста для чата

### Pipeline

Модуль Pipeline сейчас поддерживает:

- многошаговые workflows агентов
- повторы и стратегии backoff для шагов
- таймауты шагов
- условное выполнение через `runIf`
- преобразование вывода и экспорт переменных
- лимиты по общему времени, общему числу токенов и количеству шагов
- Mermaid preview и экспорт исходника
- историю запусков и детали шагов
- сохранение, импорт и экспорт

Чат также поддерживает команды `/pipeline`, например:

- `/pipeline list`
- `/pipeline run <name-or-id>`
- `/pipeline status`
- `/pipeline history <name-or-id>`
- `/pipeline cancel`

### Timer

Текущие типы таймеров:

- Once
- Interval
- Cron

Текущие действия таймера:

- уведомление на рабочем столе

## 10. Устранение неполадок

### Не удается подключиться к модели

Проверьте по порядку:

1. API key корректен
2. Base URL соответствует провайдеру
3. включена хотя бы одна модель
4. прокси не блокирует запрос
5. тест подключения в Models проходит успешно

### Канал не получает сообщения

Проверьте по порядку:

1. канал включен
2. reply-agent существует и включен
3. локальный channel server запущен для webhook-каналов
4. callback URL на стороне платформы точно совпадает с URL Suora
5. текущий чат не блокируется через `allowedChats`
6. в разделах Health или Debug нет ошибки credential

### Skill не выглядит активной

Проверьте по порядку:

1. skill включена
2. нужная skill назначена агенту
3. skill импортирована в текущий workspace или внешний каталог
4. содержимое является корректным `SKILL.md`

### Таймер не срабатывает

Проверьте по порядку:

1. таймер включен
2. Cron-выражение валидно
3. целевой агент или pipeline все еще существует
4. настольное приложение запущено

## 11. Рекомендуемый первый сценарий

Если вы только начинаете работать с текущей сборкой, удобен такой порядок:

1. добавьте провайдера и включите модель в `Models`
2. просмотрите встроенных агентов в `Agents`
3. начните первый диалог в `Chat`
4. создайте группу документов в `Documents`
5. сохраните workflow из двух или трех шагов в `Pipeline`
6. запланируйте его через `Timer`
7. настройте `Channels` или `MCP` только после того, как локальный сценарий станет стабильным
